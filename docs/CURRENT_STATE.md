# 当前项目状态

更新日期：2026-07-20

## 产品定位

“棋路”目前是一款面向零基础学习者的浏览器国际象棋陪练。玩家无需注册，输入昵称后执白棋对电脑下棋，数据仅保存在当前浏览器。

当前版本已经是可用的电脑陪练和基础规则学习工具，但“AI 教练”仍以模板讲解为主，尚未真正解释复杂走法的具体原因。现阶段不应把它描述为能够可靠理解玩家或完整讲解战略的教练。

## 已实现功能

### 棋局与棋盘

- 使用 `chess.js` 判断合法走法、将军、将死、和棋、升变、吃过路兵和王车易位。
- 支持点击和拖动走棋、合法落点提示、上一步高亮、将军提示和棋盘翻转。
- 棋盘格保持固定 8×8 尺寸，四边显示明显坐标。
- 棋子悬停 1 秒显示名称，显示 2 秒后自动消失；移开时立即隐藏。
- 新手引导最后一步提供“不再显示”，选择会随当前玩家档案保存在本地，刷新页面后继续生效。
- 电脑落子前显示起点棋子和目标格动画，落子后显示明显反馈。
- 玩家首次输入昵称时可选择男生、女生或不告诉，分别显示彩色 `👨`、`👩`、`🧑`；电脑和教练使用统一头像组件。

### 电脑引擎

- 默认使用浏览器本地 Stockfish 18 Lite 单线程 WASM。
- Stockfish 当前设置为最低限制强度，约 1320 Elo，每步搜索约 350ms。
- 玩家可以在“小白棋友 · 轻松模式”“成长棋友 · 练习模式”和“进阶棋友 · 挑战模式”之间切换，选择保存在本地。
- 成长棋友使用独立的 `GrowingOpponentEngine`：在启蒙评分上计算玩家下一步的最佳回应，能处理一步将杀和多数立即送子，但会在分数接近的安全候选中保留随机性。
- 进阶棋友底层仍为 Stockfish 18 Lite，小白棋友底层仍为 `BeginnerOpponentEngine`；新玩家继续默认进阶棋友。
- Stockfish 加载失败、超时或浏览器不支持 WASM 时自动降级到 `BeginnerOpponentEngine`。
- Stockfish adapter 会解析 `depth`、白方视角 `score`、`pv` 和最多 3 个候选着法，分析过程可以取消；浏览器不支持或引擎失败时仍会安全降级。

### 连续对话与本地教练

- 右侧已改为连续对话：依次显示玩家走法、电脑思考/分析、电脑实际落子和教练评论。
- 可展开查看 Stockfish 搜索深度、评分和主要变化；动态分析消息不会写入存档，完成的走法与教练消息会保存。
- SAN 与白话说明显示在同一行；吃子会明确说明移动到哪个格子以及吃掉哪枚棋子。推荐走法由 Stockfish 对玩家当前局面重新搜索，并展开为棋子、起点和终点。
- 教练可以关闭；关闭后电脑陪练与走法消息仍正常工作。
- 能展开 SAN 基础含义，并能专门解释 `O-O`、`O-O-O`、`+` 和 `#`。
- 教练内容随当前时间线节点保存，F5 刷新和悔棋/取消悔棋后可以恢复。
- 默认使用 `LocalCoachService`，不需要网络或 API 费用。
- 可通过 `NEXT_PUBLIC_COACH_API_URL` 接入外部教练网关，失败时回退本地教练。
- 本地解释已经能识别吃子、将军/将死、易位、发展、占据中心、打开线路和新产生的攻击，并在证据不足时明确降级。
- 当前主要不足：解释器还不能稳定识别钉住、串击、复杂防守目的和多步战术；三级建议目前仍是基于同一回合上下文的简化版本。

### 复盘与本地数据

- localStorage 保存昵称、匿名 playerId、语言、引擎/教练选择、学习进度、棋局和最多 300 条完成的对话消息。
- 时间线最多保留初始节点加 100 个完整回合节点。
- 玩家可以连续悔棋和取消悔棋；在历史位置重新走棋后，未来分支被截断。
- 兼容没有 `crypto.randomUUID` 的旧版或非安全上下文浏览器。

### 国际化与部署

- 支持简体中文、英文和日文。
- 首次输入昵称的欢迎弹窗也提供语言选择，不创建玩家档案即可立即切换界面语言。
- Next.js 静态导出，不需要 Node.js 应用服务器。
- `npm run build:server` 生成根路径版本供 `chess9527.com`/Nginx 使用；`npm run build:pages` 生成 `/ChessClass` 版本供 GitHub Pages 使用，`npm run build` 默认指向服务器版本。
- `deploy/nginx/chess9527.conf` 提供自有服务器生产配置：`.com` 为主域名，`.online`/`www` 跳转，支持静态路由、Stockfish WASM、缓存、安全响应头和地区日志字段。
- GitHub Pages 已部署：`https://john04047210.github.io/ChessClass/`。
- GitHub Actions 使用 Node.js 24；当前工作流只允许手动触发发布，普通 `push` 不会自动部署。
- 普通构建约 8.5 MB，其中 Stockfish Lite WASM 约 7 MB。
- 内置三语规则和技巧弹窗，可在对局中随时查看；内容集中维护并使用应用自己的棋盘与棋子样式。
- 使用独立 SVG 马头站点图标，适合 GitHub Pages 和普通服务器部署。
- 浏览器不包含访问统计 SDK 或统计上报。未来自有服务器通过 access log 做访问量和粗略地区聚合；准确停留时间暂不统计。
- 当前没有需要服务器保护的公开写接口，因此不加载 CAPTCHA；AI、账号或在线功能出现后再按地区和风险接入并执行服务端验票。

## 测试基线

- Vitest：6 个测试文件、31 个单元测试。
- Playwright：12 个核心浏览器场景。
- 覆盖三种语言、Stockfish 真实走棋、引擎降级、引擎选择持久化、刷新恢复、悔棋分支和旧浏览器 UUID。
- 最近增加棋子提示 2 秒自动隐藏的浏览器验证。

测试数量会随功能变化；以当前测试输出为准，不应只依赖本文数字。

## 当前主要限制

- 局面解释仍是规则型分析，面对复杂战术时不能达到真人教练或大模型的语言理解能力。
- 已提供三档电脑对手，但尚未经过足够真人对局校准出稳定的近似等级；Stockfish 候选变化也尚未完整参与教练的自然语言解释。
- 玩家固定执白，尚无执黑或随机分配。
- 没有账号、云同步、跨设备恢复、真人对战、排行榜或支付。
- 外部 AI 网关只有客户端契约，仓库内没有可部署的网关服务。
- localStorage 被清除后无法恢复数据。
