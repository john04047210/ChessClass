# 局面感知型本地教练草案

- 状态：待评审
- 优先级：P0
- 背景：当前 `LocalCoachService` 大量使用固定模板，玩家无法从“发展棋子、争夺空间”理解电脑的具体考虑。

## 目标

每个完整回合结束后，向完全零基础玩家提供简短、具体、能在棋盘上验证的说明。基础功能必须在浏览器本地运行，不依赖远程 AI。

## 非目标

- 不保证还原棋手或引擎唯一的心理动机。
- 第一阶段不解释深层残局战略、长线弃子或复杂兵形理论。
- 不把 Stockfish 评价包装成绝对真理。
- 不要求换用完整 100 MB Stockfish。

## 玩家需要得到的答案

1. 刚才发生了什么？
2. 电脑这一步具体改变了什么？
3. 这对我有什么直接影响？
4. 下一步应该观察哪个棋子、格子或威胁？

## 分析输入

- 用户走棋前 FEN。
- 用户走棋后、电脑走棋前 FEN。
- 电脑走棋后 FEN。
- 用户与电脑的 MoveRecord。
- Stockfish 的 bestmove、score、depth 和 PV。
- 可选候选着法及评分差距。

## 可确定的棋盘事实

- 吃子、将军、将死、易位、升变。
- 走前被攻击的棋子是否移动或获得保护。
- 走后新增攻击和保护关系。
- 是否打开或关闭象、车、后的直线。
- 马或象是否首次离开初始位置。
- 是否占据中心格或改变对中心格的控制。
- 是否直接回应用户上一手产生的威胁。
- 下一回合是否存在立即吃子、将军或必须应对的威胁。

## 推测性原因

以下内容必须标记为“可能”，并附带计算依据：

- 根据 PV 推测下一步准备。
- 根据候选着法评分差推测为何选择当前着法。
- 根据长期位置变化推测战略目的。

无法获得足够证据时，应说“这一步暂时没有明显的直接战术目的”，然后描述确定事实，不能填入万能模板。

## 建议输出结构

```ts
interface LocalMoveAnalysis {
  facts: Array<{ key: string; text: string }>;
  hypotheses: Array<{ key: string; text: string; confidence: "low" | "medium" | "high"; evidence: string[] }>;
  immediateThreats: Array<{ square?: string; piece?: string; text: string }>;
  nextObservation: { text: string; squares: string[] };
  engine?: { depth?: number; scoreCp?: number; mateIn?: number; pv?: string[] };
}
```

最终三语文案应由结构化 key 和参数生成，不在分析器里硬编码完整中文句子。

## 示例

若电脑走 `d7-d6`，系统只有在验证后才能输出类似：

> 黑兵从 d7 到 d6。这一步打开了 c8 象的斜线。根据电脑计算的后续变化，它可能准备推进 e5；目前没有直接攻击你的棋子。下一步留意 e5 格和 c8 的象。

如果没有发现上述证据，则不应声称它在准备 e5。

## 分阶段实现

### 阶段 A：Stockfish 分析数据

- 扩展 Worker 消息解析，保存最近一次搜索的 depth、score 和 PV。
- 为超时、无 score、mate score 和升变 UCI 着法建立测试。
- 不改变现有 bestmove 和降级行为。

### 阶段 B：棋盘差异分析器

- 建立纯函数模块，输入走棋前后 FEN 与 MoveRecord，输出结构化事实。
- 为每类事实建立最小棋局单元测试。
- 优先覆盖吃子、将军、易位、发展、中心、开线、攻击和保护。

### 阶段 C：新手解释组合器

- 将事实按教学价值排序，默认只显示一到两个关键原因。
- 第 4 项输出具体观察点和可选棋盘格高亮。
- 三语同步，保持句子短且不依赖专业术语。

### 阶段 D：可选 AI 增强

- 将结构化分析发送给可选网关进行润色或回答追问。
- 提示词明确禁止更改棋盘事实。
- 网关失败时本地解释完整可用。

## 验收标准

- 普通兵步、发展、吃子、将军、易位和躲避攻击产生不同的具体解释。
- 第 3 项不再对所有着法复用同一句原因。
- 第 4 项至少指出一个具体棋子、格子或迫近威胁；没有依据时明确说明。
- 每个事实均可由 FEN、MoveRecord 或 Stockfish 输出验证。
- 推测文本包含不确定措辞和证据来源。
- 三种语言结构一致。
- Stockfish 不可用时仍能使用 chess.js 棋盘事实和启蒙引擎完成基础解释。
- 保持静态导出、GitHub Pages 和现有 localStorage 存档兼容。
