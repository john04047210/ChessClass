# 成长棋友引擎

## 目标

在“小白棋友 · 轻松模式”和“进阶棋友 · 挑战模式”之间提供稳定的中间棋力，让已经会走棋但经常输给专业引擎的玩家获得可练习、可获胜的对手。

## 非目标

- 不替代 Stockfish，也不追求专业棋力。
- 不增加网络服务、WASM 资源或第三方引擎依赖。
- 不让成长引擎承担自然语言教练职责。
- 不修改 `chess.js` 的规则事实边界。

## 棋力边界

成长引擎在现有小白引擎的直观原则上增加一层玩家回应计算：

1. 为电脑的每个合法走法建立局面。
2. 枚举玩家下一步的合法回应，并按玩家最有利的回应评价电脑走法。
3. 识别一步将杀、直接吃子、立即送子、基本子力、中心、出子、易位和王的安全。
4. 从分数接近的安全候选中按固定种子加权选择，不保证每次走理论最佳着法。

这使它能够避免多数“一步送后/送车”和明显将杀，同时仍会在位置选择和多步战术上犯错。

## 产品行为

- 三档依次显示为“小白棋友 · 轻松模式”“成长棋友 · 练习模式”“进阶棋友 · 挑战模式”。
- 新玩家继续默认选择进阶棋友；玩家可以手动选择成长棋友，已有玩家保留原选择。
- 选择保存在现有 `PlayerProfile.opponentEngine` 字段中。
- Stockfish 加载失败仍降级到小白棋友，成长棋友本身不依赖 Stockfish。
- 推荐走法继续优先使用 Stockfish 分析，不受电脑对手档位影响。

## 兼容性

- 旧存档中的 `stockfish` 和 `starter` 保持原义。
- 缺失或非法的引擎值按现有规则迁移为 `stockfish`。
- 三种语言保持同样的三个档位。

## 验收标准

- 成长引擎始终返回 `chess.js` 认可的合法着法。
- 固定种子下结果可复现。
- 无合法着法时返回 `null`。
- 能选择一步将杀，并能避免存在安全替代时的一步重大送子。
- 玩家可以选择三个档位，刷新后选择仍保留。
- 玩家可以选择成长模式并在刷新后保留，旧的两种合法选择不会被覆盖。
- 静态构建和三语核心流程通过。
