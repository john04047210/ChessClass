# 服务器侧地区与访问统计方案

更新日期：2026-07-20

## 已确定原则

- 浏览器不加载 Google Analytics 或其他统计 SDK，不发送访问、头像、昵称、棋局或停留时间事件。
- 当前前端不存在统计开关、统计 visitorId 或统计 localStorage。
- 访问量、来源、状态码、User-Agent 和粗略国家/地区只从反向代理、CDN 或 Web 服务器日志分析。
- 原始访问日志设置短保留期并限制访问；长期报表只保存按日、国家/地区和页面聚合后的数字。
- 头像性别只用于本地显示，不进入服务器统计。

## 可从访问日志获得

- 请求数、独立 IP 近似访客数、热门页面、Referer、设备/User-Agent、状态码和流量。
- 使用 CDN 国家代码或离线 GeoIP 数据库，可以把请求 IP 转换为国家/地区；聚合后不保留原始 IP。
- 机器人会污染请求量，应分别统计已知爬虫、异常高频 IP 和正常浏览器请求。

独立 IP 不是严格的独立用户：公司、学校和移动网络可能共享 IP，同一用户也可能更换 IP。因此报表应写“估算访客”，不能把它当账号数。

## 停留时间的边界

仅凭 access log 通常只能用“同一匿名访问来源相邻请求的时间差”估算多页面会话；当前应用是单页棋局，用户长时间操作时可能没有新的 HTML 请求，所以日志无法得到可靠停留时间。

若以后确实需要停留时间，应单独评审一个第一方、最小化的浏览器事件方案：只在页面可见状态变化时发送累计秒数，不发送棋局内容，并提供隐私说明。当前版本不实现。

## 防机器人策略

当前昵称、走棋和本地存档都不访问服务器，不需要 CAPTCHA。未来只保护会消耗服务器资源或改变服务器状态的端点，例如：

- AI 教练请求；
- 注册、登录和找回密码；
- 评论、反馈或在线对战房间；
- 任何可写入数据库的接口。

推荐顺序：限流与请求大小限制 → WAF/机器人规则 → 蜜罐和行为阈值 → 只对可疑请求弹 CAPTCHA → 服务端验票。仅在浏览器显示验证码但不做服务端验票没有安全作用。

### 分区验证码

- 中国大陆：优先评估腾讯云验证码或阿里云 CAPTCHA，它们都要求客户端获取票据并由业务服务器验证。
- 其他地区：可以评估 Google reCAPTCHA。
- Google 官方提供在无法连接 `google.com` 时改用 `recaptcha.net` 的加载方式，但不能据此承诺中国大陆所有网络稳定可用。
- 不建议在浏览器先调用第三方定位再选择验证码。应由服务器/CDN 国家代码选择供应商，并始终提供失败回退，避免真实玩家被锁在页面外。

## 部署时再实现

当前仓库没有统计后端、验证码密钥或需要保护的公开写接口，因此不加入任何验证码脚本。服务器部署方案确定后，需要新增：日志格式与脱敏轮转、GeoIP 聚合任务、机器人过滤规则、供应商路由、验证码服务端验票和监控。
