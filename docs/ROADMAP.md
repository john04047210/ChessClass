# 产品路线图

路线图描述优先级，不承诺具体发布日期。完成工作后应同步更新 `CURRENT_STATE.md`，重大方向变化应更新 `DECISIONS.md`。

## P0：局面感知型本地教练

目标：让产品从“电脑陪练”提升为真正对新手有帮助的教学工具。

- 扩展 Stockfish adapter，解析 `depth`、`score`、`pv`，必要时支持多个候选着法。
- 比较电脑走棋前后的局面，识别吃子、将军、将死、易位、升变、躲避攻击、保护棋子、制造攻击、打开线路、发展棋子和中心影响。
- 第 3 项“电脑为什么这样走”只输出有证据的具体原因。
- 第 4 项“下一步观察什么”指出具体棋子、格子或威胁。
- 分开显示确定事实与推测，并为推测提供置信度。
- 复杂局面无法可靠解释时诚实降级，不使用“发展棋子、争夺空间”万能模板。
- 保持完全本地、静态和离线可用，远程 AI 仅用于可选润色和追问。

局面解释细节见 `docs/features/position-aware-local-coach.md`；右侧真人式连续对话与完整实施顺序见 `docs/features/conversational-coach-v3.md`。

## P1：难度与教学节奏

- 增加清晰的难度选择，而不是只显示固定约 1320 Elo。
- 保留启蒙引擎作为最低难度，Stockfish 提供多个受限级别。
- 根据学习者水平控制说明长度、术语数量和提示强度。
- 避免电脑在极低水平仍连续走出对新手压迫过强的着法。

## P1：复盘体验

- 在时间线上显示双方每回合着法和关键事件。
- 悔棋浏览时同步显示当时的原因分析和下一步提示。
- 增加对局结束总结：关键转折、已学规则和一个最值得改进的习惯。
- 支持导出 PGN；导入功能需先明确安全和数据兼容策略。

## P2：结构化课程

- 用短实战目标覆盖走子、吃子、将军、将死、易位、升变和基本开局原则。
- 课程进度继续本地保存。
- 教学目标与自由对局分离，避免自由对局提示过度干扰。

## P2：可选远程 AI

- 定义稳定的教练网关协议、鉴权、预算、缓存和限流要求。
- 远程 AI 接收本地结构化分析结果进行表达增强，不让模型凭空推断棋盘事实。
- 明确玩家数据发送范围和隐私提示。
- 浏览器静态客户端不直接内置共享 API Key。

## P3：人人对战

- 先支持同一设备本地双人，验证颜色、座位、对话角色和时间线不再依赖“玩家固定执白、电脑固定执黑”。
- 在线模式单独设计账号或匿名身份、邀请房间、权威棋局服务、走法版本、断线重连和双方同意悔棋。
- 在线实时对局默认关闭教练提示，避免形成作弊；赛后复盘可以重新启用 Stockfish 和教练。
- 复用本次对话模型，但引擎候选只在人机或赛后分析中展示。
- 当前阶段只预留参与者和走棋来源抽象，不引入 WebSocket、数据库或强制后端。

## 暂不优先

- 真人实时对战（已列入 P3，本阶段不实现）。
- 排行榜、社交和支付。
- 强制注册或云同步。
- 为追求最高棋力换用 100 MB 以上的完整 Stockfish。
